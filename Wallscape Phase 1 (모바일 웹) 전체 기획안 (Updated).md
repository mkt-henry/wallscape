# Wallscape Phase 1 (모바일 웹) 전체 기획안 (Updated)

본 문서는 그래피티 작품 공유 SNS 'Wallscape'의 초기 시장 검증 및 MVP(Minimum Viable Product) 출시를 위한 **Phase 1 (모바일 웹/PWA)** 전체 기획안입니다. 확정된 최신 기술 스택(Vercel + Supabase + Kakao Maps)을 반영하여 업데이트되었습니다.

---

## 1. 서비스 개요 및 비전

### 1.1 기획 배경 및 목적

그래피티(Graffiti)는 도시의 벽면이나 공공 공간에 그려지는 거리 예술로, 시간이 지나면 훼손되거나 사라지는 일시적인 특성을 가집니다. 이러한 작품들은 지역의 문화적 정체성을 반영하는 중요한 예술적 기록임에도 불구하고, 이를 체계적으로 보존하고 공유할 수 있는 전용 플랫폼이 부재한 상황입니다.

Wallscape는 누구나 스마트폰으로 거리의 그래피티 작품을 촬영하고 업로드하여 디지털 아카이브를 구축하는 **위치 기반 그래피티 공유 SNS**입니다. 사진 파일에 내장된 GPS 메타데이터(EXIF)를 자동으로 인식하거나 수기로 위치를 입력하여, 작품이 실제 존재하는 위치를 지도 위에 시각화합니다. 이를 통해 도시의 숨겨진 예술 작품들을 탐색하고, 예술 애호가들이 연결되는 커뮤니티를 형성하는 것을 목적으로 합니다.

### 1.2 서비스 비전

> "모든 거리의 벽을 세계 최대의 야외 미술관으로 만든다."

### 1.3 타겟 사용자 (Target User)

| 사용자 유형 | 특징 | 주요 니즈 |
|---|---|---|
| **그래피티 아티스트** | 자신의 작품을 세상에 알리고 싶은 창작자 | 작품 포트폴리오 구축, 팬과의 소통 |
| **스트리트 아트 팬** | 그래피티 및 거리 예술에 관심이 많은 감상자 | 새로운 작품 발견, 아티스트 팔로우 |
| **도시 탐험가 / 여행자** | 새로운 도시나 동네의 숨겨진 명소를 찾는 사람 | 지역별 작품 지도 탐색, 방문 리스트 생성 |
| **사진 애호가** | 도시 풍경과 예술 작품을 사진으로 기록하는 사람 | 사진 공유, 커뮤니티 반응 수집 |

## 2. 핵심 기능 명세 (Phase 1 기준)

Phase 1은 모바일 웹 브라우저 환경에서 동작하며, 서비스의 핵심 가치인 '위치 기반 아카이빙'과 '지도 탐색'에 집중합니다.

### 2.1 사진 업로드 및 위치 정보 처리 (핵심 차별화 요소)

사용자가 그래피티 작품 사진을 업로드할 때, 브라우저 단에서 위치 정보를 빠르고 직관적으로 처리합니다.

- **EXIF GPS 메타데이터 자동 추출**: 클라이언트 사이드(브라우저)에서 `exif-js` 등의 라이브러리를 사용하여 업로드된 사진의 EXIF 데이터를 즉시 파싱합니다. GPS 정보가 존재할 경우, **Kakao 로컬 API**의 역지오코딩을 통해 좌표를 국내 주소로 자동 변환하여 입력 폼에 채워줍니다.
- **수기 위치 입력 및 핀 조정**: 사진에 GPS 정보가 없거나 위치가 부정확한 경우, 사용자가 직접 주소를 검색하거나 지도 위에서 핀을 드래그하여 정확한 위치를 수동으로 지정할 수 있습니다.

### 2.2 인터랙티브 지도 뷰 (Map View)

사용자들이 업로드한 모든 그래피티 작품의 위치를 지도 위에 마커로 시각화합니다.

- **마커 및 클러스터링**: 각 게시물의 대표 사진 썸네일을 마커로 표시하며, 지도를 축소할 경우 인접한 마커들을 그룹으로 묶어 숫자와 함께 표시(클러스터링)하여 렌더링 성능을 확보합니다.
- **마커 미리보기 (Bottom Sheet)**: 지도에서 특정 마커를 클릭하면 화면 하단에서 팝업이 올라와 작품 사진, 제목, 주소 등을 미리 보여줍니다.
- **현재 위치 기반 탐색**: HTML5 Geolocation API를 활용하여 사용자의 현재 위치를 파악하고, 주변의 그래피티 작품을 탐색할 수 있도록 지원합니다.

### 2.3 피드 및 소셜 인터랙션

인스타그램과 유사한 직관적인 카드형 피드 인터페이스를 제공합니다.

- **피드 정렬**: 최신순, 인기순(좋아요/댓글 기반), 내 주변순, 팔로잉 기준으로 피드를 정렬하여 감상할 수 있습니다.
- **소셜 기능**: 마음에 드는 작품에 '좋아요'를 누르고, 댓글을 통해 소통하며, 관심 있는 사용자를 '팔로우'할 수 있습니다. 또한, 나중에 다시 보고 싶은 작품은 '저장(북마크)'할 수 있습니다.

## 3. 페이지 구성 및 정보 구조 (Information Architecture)

Phase 1 모바일 웹 환경에 맞춘 전체 페이지 구성안입니다. 하단 고정 네비게이션 바(Bottom Tab Bar)를 중심으로 5개의 주요 탭으로 구성됩니다.

### 3.1 공통 및 인증 (Auth)
- **스플래시 / 온보딩**: 서비스 로고 노출 및 주요 기능(지도 탐색, 업로드) 소개
- **로그인 / 회원가입**: 이메일 로그인 및 소셜 로그인 (Google, Kakao, Apple) 지원

### 3.2 메인 네비게이션 (Bottom Tab Bar)

| 탭 이름 | 하위 페이지 및 주요 기능 |
|---|---|
| **1. 홈 (Feed)** | **피드 메인**: 최신순/인기순/내 주변순 피드 목록<br>**게시물 상세**: 고해상도 사진, 미니 지도, 본문, 좋아요/댓글<br>**댓글 전체보기**: 댓글 및 대댓글 목록 |
| **2. 지도 (Map)** | **지도 메인**: 전체 화면 지도, 마커 클러스터링, 내 위치 이동<br>**마커 미리보기**: 하단 Bottom Sheet 형태의 요약 정보 |
| **3. 업로드 (Upload)** | **사진 선택**: 디바이스 갤러리 접근 및 다중 선택<br>**위치 설정**: EXIF 자동 추출 결과 확인 및 지도 핀 수동 조정<br>**정보 입력**: 제목, 설명, 해시태그, 아티스트 태그 작성 |
| **4. 알림 (Activity)** | **알림 메인**: 좋아요, 댓글, 팔로우 등 소셜 활동 알림 내역 |
| **5. 프로필 (Profile)** | **내 프로필**: 내 정보, 업로드한 작품 갤러리, 저장한 작품 목록<br>**타인 프로필**: 타 사용자 정보 및 작품 갤러리<br>**설정**: 계정 관리, 알림 설정, 로그아웃 |

### 3.3 검색 (Search)
- **검색 메인 및 결과**: 해시태그, 지역명, 사용자 계정 기반의 통합 검색 기능 제공

## 4. 기술 스택 및 아키텍처 (Phase 1 확정안)

초기 시장 검증을 위한 웹 기반 아키텍처이며, 서버리스 환경(Vercel)과 BaaS(Supabase)를 결합하여 인프라 관리 부담을 최소화하고 개발 속도를 극대화하도록 설계되었습니다.

### 4.1 프론트엔드 및 백엔드 (Vercel Serverless)
- **프레임워크**: **Next.js (React)** + **TypeScript**
  - 프론트엔드 렌더링과 백엔드 API(API Routes / Server Actions)를 단일 프로젝트에서 처리합니다.
- **상태 관리 및 스타일링**: **Zustand**, **React Query**, **Tailwind CSS**
- **지도 및 위치**: **Kakao Maps JavaScript API**
  - 국내 주소 정확도가 가장 높고 무료 한도가 넉넉하여 국내 타겟 서비스에 최적화되어 있습니다.
- **배포 및 호스팅**: **Vercel**
  - 별도의 서버 인스턴스 없이 서버리스 형태로 자동 스케일링됩니다.

### 4.2 데이터베이스 및 인증 (Supabase)
- **데이터베이스**: **PostgreSQL** + **PostGIS** (Supabase 내장)
  - 위치 기반 서비스의 핵심입니다. 위도/경도를 `GEOMETRY(Point)` 타입으로 저장하고, `ST_DWithin` 공간 함수를 사용하여 "반경 N km 이내 작품 검색"을 인덱스 기반으로 초고속 처리합니다.
- **DB 연결 풀링**: **Supavisor (Transaction Mode)**
  - Vercel 서버리스 함수의 특성상 발생하는 DB 커넥션 고갈 문제를 해결하기 위해 Supabase가 제공하는 내장 커넥션 풀러를 반드시 사용합니다.
- **인증**: **Supabase Auth**
  - 이메일 로그인 및 OAuth(Google, Kakao, Apple) 소셜 로그인을 손쉽게 구현합니다.

### 4.3 스토리지 및 캐싱
- **이미지 스토리지**: **Supabase Storage**
  - Vercel 서버리스 함수의 응답 크기 제한(4.5MB)을 우회하기 위해, 클라이언트에서 API를 거치지 않고 Supabase Storage로 직접 이미지를 업로드(Presigned URL 방식)합니다.
- **캐싱**: **Vercel KV (Redis)**
  - 인기 피드, 세션 데이터 등 빈번하게 조회되는 데이터를 캐싱하여 DB 부하를 줄입니다.

## 5. 향후 발전 방향 (Phase 2 로드맵)

Phase 1(모바일 웹)을 통해 시장성을 검증한 후, 다음과 같은 네이티브 앱(Phase 2) 전용 기능으로 확장할 계획입니다.

- **플랫폼 전환**: React Native (Expo)를 도입하여 iOS 및 Android 네이티브 앱 출시 (백엔드/DB는 기존 Supabase 그대로 재사용)
- **지도 성능 극대화**: `react-native-maps`를 활용한 네이티브 지도 엔진 렌더링으로 압도적으로 부드러운 패닝/줌 경험 제공
- **네이티브 푸시 알림**: Firebase Cloud Messaging(FCM)을 연동하여 이미지가 포함된 풍부한 형태의 OS 레벨 푸시 알림 완벽 지원
- **카메라 제어 강화**: 네이티브 카메라 및 갤러리 API를 직접 호출하여 더 안정적인 EXIF 데이터 추출 및 다중 이미지 선택 UI 제공
